# English + Spanish (i18n)

Bilingual sites, per-client opt-in. English lives at the root; Spanish
mirrors every path under `/es` — `/menu` → `/es/menu`.

## Turning it on

```ts
// config/site.ts
locales: ["en", "es"],
```

That single line enables: the `/es` mirror (middleware rewrite + `x-locale`
header), the header language switcher, `hreflang` alternates on every page,
`/es` URLs in the sitemap, `<html lang>`, and the Spanish UI dictionary.
English-only clients (omit the field) see zero change.

## The three layers of translation

1. **Framework UI** (buttons, form labels, booking slots, 404s, nav
   titles): shipped in `lib/i18n/dictionaries/es.ts`, typed against the
   English dictionary — a missing key is a compile error. Standard nav
   titles translate via `lib/i18n/nav.ts`; custom titles fall back to
   English (add them to the map).
2. **Form definitions**: each `FormDef` in `config/forms.ts` has an `es`
   block (all seven shipped forms are fully translated). Validation
   messages, success messages, and autoresponders follow the visitor's
   language; the **owner notification stays English**.
3. **Database content** (services, promotions, team bios, location intros,
   service areas): the `translations` table (migration 0010) overlays
   Spanish field-by-field at render time, falling back to English per field
   — a half-translated site shows mixed content, never holes.

## Translating content

AI Studio → "Translate content to Spanish": pick a content type, click, and
missing translations fill in batches of 8 (re-run until done). Neutral
Latin American Spanish; proper nouns and addresses are preserved. Review on
the `/es` site. `TRANSLATABLE_FIELDS` in `lib/i18n/content.ts` is the
single source of truth for what translates.

## For developers

- Public components link with `LocalLink` (drop-in for `next/link`) so
  visitors stay in their language; admin uses plain `Link` (dashboard is
  English by design).
- Server code: `const { locale, dict } = await getDict()`. Client
  components receive resolved strings as props (the context carries only
  the locale tag).
- Service-area page titles use the module's `serviceNounEs`
  ("Techado en Jupiter, FL") — all nine trades ship with one.

## Known limits (backlogged)

Blog posts and CMS page blocks don't translate yet (long-form content
wants side-by-side editing, not field overlay). Canonicals point to the
English URL with hreflang alternates; per-locale self-canonicals need the
const-metadata pages converted to generateMetadata.
